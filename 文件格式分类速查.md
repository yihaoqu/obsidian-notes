# 文件格式分类速查

## 0. 底层原理:电信号 → 比特 → 字节

计算机内部与线路传输的信息,本质是电信号(电压高/低、电流有/无、磁场方向)。数字电路把连续的电信号量化成两种离散状态:高电平记为 1,低电平记为 0(具体判定阈值因电路标准而异,例如 TTL 逻辑中通常约 2V 以上视为 1、0.8V 以下视为 0)。

不同存储介质把这个抽象的 1/0 状态,固化成不同的物理载体:

| 介质 | 1/0 的物理载体 |
|---|---|
| SSD(闪存) | 晶体管浮栅内是否囚禁电荷 |
| HDD(机械硬盘) | 磁性颗粒的磁场方向(N 极 / S 极) |
| RAM | 电容是否带电(需持续刷新,断电即丢失) |
| 光纤 / 网线传输 | 光脉冲的有无,或电压随时间的变化波形 |

8 个比特(bit)组成 1 个字节(byte),字节是绝大多数文件格式的最小寻址单位。

## 1. 从字节序列到"文件格式"

文件在存储介质上,本质只是一段连续的字节序列。文件格式是一套约定好的规则,规定这段字节序列该如何被拆解、赋予意义。同一段字节按不同规则解读会得到完全不同的结果——后缀名本身只是"建议按哪套规则解析",不强制内容一定符合该规则。

## 2. 按存储结构分类

| 结构类型 | 特点 | 举例 |
|---|---|---|
| 纯文本(Plain text) | 每个(或每几个)字节直接对应一个可读字符,靠字符编码表(如 UTF-8)转换 | .txt .md .csv .json .yaml .xml .html .css .py .java .c |
| 结构化二进制(Binary) | 字节按专有协议编排(header + 数据块 + 校验位等),需专用程序解码,不可直接读出 | .png .jpg .mp3 .mp4 .exe .sqlite .ttf |
| 压缩包容器(Zip-based container) | 本质是一个 zip 压缩包,内部装有若干 XML / 二进制文件 | .docx .xlsx .pptx .apk .jar .epub |
| 复合二进制容器(Compound binary) | 早期微软专有格式,近似"文件系统套文件系统" | .doc .xls .ppt(97-2003 格式) |

## 3. 按内容用途分类

### 3.1 文本 / 标记 / 数据交换
- `.txt` — 无格式纯文本
- `.md` — Markdown,带排版语法的纯文本
- `.csv` — 逗号分隔的表格数据
- `.json` — 键值对结构化数据,程序间数据交换常用
- `.yaml` / `.yml` — 结构化配置,较 JSON 更适合人工阅读
- `.toml` — 配置文件格式
- `.xml` — 标签嵌套的结构化数据
- `.html` — 网页标记语言
- `.ini` — 早期配置文件格式
- `.log` — 日志文本

### 3.2 文档
- `.pdf` — 固定页面排版,二进制结构
- `.docx` / `.doc` — Word 文档(`.docx` 为 zip 容器,`.doc` 为复合二进制)
- `.odt` — OpenDocument 文本格式
- `.rtf` — 富文本格式,跨软件兼容用

### 3.3 图片
- 位图 / 栅格(逐像素记录颜色):`.png`(无损)、`.jpg`/`.jpeg`(有损压缩)、`.gif`(支持动画,颜色数有限)、`.bmp`(未压缩)、`.webp`(现代压缩格式)、`.tiff`(高质量印刷用)
- 矢量(记录坐标与路径公式,不逐像素记录):`.svg`(基于 XML 的文本格式)、`.ai`、`.eps`

### 3.4 音频
- `.mp3` — 有损压缩,兼容性最广
- `.wav` — 无损,体积大
- `.flac` — 无损压缩
- `.aac` — 有损,常见于流媒体
- `.ogg` — 开源有损格式

### 3.5 视频
- `.mp4` — 最通用的容器格式
- `.mkv` — 开源容器,支持多音轨 / 字幕
- `.avi` — 早期容器格式
- `.mov` — QuickTime 容器
- `.webm` — 网页优化的开源格式

### 3.6 压缩 / 归档
- `.zip` — 最通用的压缩格式
- `.rar` — 高压缩率,专有格式
- `.7z` — 高压缩率,开源
- `.tar` — 打包不压缩(常与 `.gz` 组合成 `.tar.gz`)
- `.gz` — Gzip 压缩

### 3.7 可执行程序
- `.exe` — Windows 可执行文件
- `.app` — macOS 应用程序包(本质是文件夹)
- `.apk` — Android 安装包(zip 容器)
- `.dll` / `.so` — 动态链接库(Windows / Linux)
- `.class` — Java 编译后的字节码

### 3.8 字体
- `.ttf` — TrueType 字体
- `.otf` — OpenType 字体
- `.woff` / `.woff2` — 网页专用字体压缩格式

### 3.9 数据库
- `.sqlite` / `.db` — 轻量级嵌入式数据库文件
- `.mdb` — Access 数据库

## 4. 记忆锚点

- 后缀名是建议,不是强制
- 纯文本:字节直接映射字符,任何编辑器都能打开
- 二进制:字节按专有协议编排,需专用程序解码
- 所有格式最终都能还原成 0/1 序列,格式的区别仅在于这串 0/1 该如何分段、如何解释
